# Workflow Skill

完整工作流指引：计划模式、任务分工、算法验证、验收交付、交互规范。

## Instructions

### 1. 计划模式
- 进入后**只读不写**，研究代码
- 思考完**输出方案到屏幕**，等用户同意
- 用户说可以后才调 ExitPlanMode 执行

### 2. 任务分工
- **简单几行修改** → 直接自己 edit
- **多文件修改** → 派子 agent（model: haiku）执行，自己验收
- 调用 Agent 工具时加 `model: "haiku"`

### 3. 算法改动（修改-验证-交付）
- **先在 /tmp 写验证程序**，构造具体数值例子验证算法
- 验证通过后才改正式代码
- 正式代码中**加日志**（RCLCPP_DEBUG/INFO）辅助确认
- 验证完**删除 /tmp 临时文件**

### 4. 验收流程
- 检查关键文件内容是否正确
- `colcon build` 编译通过
- grep 确认无残留引用
- 不符合预期 → 让子 agent 重写

### 5. 交付
- 编译通过 + 逻辑正确 → 交付
- 有问题 → 停下来报告用户

### 6. 交互规范
- 用 AskUserQuestion 呈现选择题，不要手动输入字母
